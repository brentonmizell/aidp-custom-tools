# Custom Tools for AIDP Agent Flows

A curated set of Custom Code tools for AIDP (Oracle AI Data Platform) Agent
Flows, plus a template and a per-tool testing kit.

## Quick start (interactive)

```
git clone <this repo>
cd aidp-custom-tools
python setup.py        # runs the wizard: init -> configure -> build
```

That's it. The wizard:

1. Creates `~/.aidp/aidp-deploy.config.json` if missing (prompts for region,
   data lake OCID, workspace ID).
2. Verifies `~/.oci/config` exists for tenancy / user / fingerprint auto-fill.
3. Walks every tool's `conf` and fills in workspace-specific values
   (catalog / schema / volume / KB key / etc.) using the inline `_confDescriptions`
   to tell you exactly where to find each value.
4. Rebuilds every `.zip` ready to upload via the AIDP console.

## setup.py subcommands

```
python setup.py wizard      # default: init -> configure -> build
python setup.py init        # generate / verify ~/.aidp + ~/.oci config
python setup.py configure   # walk every tool's conf and fill values
python setup.py new-tool    # scaffold a brand-new custom tool package
python setup.py build       # auto-fill + rebuild zips (non-interactive)
python setup.py status      # show what's filled vs blank across all tools
```

### Examples

```
# First time on a new machine — full walk-through:
python setup.py

# Just check what's filled:
python setup.py status

# Skip the prompts; just auto-fill region/OCIDs and rebuild zips:
python setup.py build

# Create a new tool from scratch (scaffolds tool_config.json +
# tool_implementation.py + README + zip):
python setup.py new-tool

# Re-prompt for AIDP config values (e.g. switching workspaces):
python setup.py init --re-prompt

# Build against a different OCI profile:
python setup.py build --profile aidp-session
```

## What's here

- **`setup.py`** — interactive wizard. The recommended entry point.
- **`build_with_config.py`** — non-interactive build (equivalent to `setup.py build`).
- **`CONFIG.md`** — canonical reference for every conf field: what auto-fills
  from where, and exactly where in the AIDP / OCI console to find each value
  you have to fill yourself.
- **`UI_HINTS_SPEC.md`** — the `_uiHints` sidecar contract (drives the AIDP
  Flow Designer VS Code extension's catalog-backed dropdowns + secret masking).
- **`INDEX.md`** — catalog of every package, its tools, dependencies, and
  strategic roadmap mapping.
- **`CUSTOM_CODE_TEMPLATE/`** — starting point for building a tool by hand.
  Most people should use `python setup.py new-tool` instead.
- **`CUSTOM_CODE_TOOLS/`** — one folder per tool package. Each folder has:
    - `README.md` — what the package's tools do.
    - `<package>.zip` — the uploadable artifact (regenerated by `setup.py build`).
    - `src/` — editable source (`tool_config.json`, `tool_implementation.py`,
      `requirements.txt`, `utils/`).
    - `TESTING_INSTRUCTIONS.md` — what to upload, configure, and enter in the
      AgentFlow Test panel, with expected results.
    - `mock_files/` — sample data to paste or upload for the tests.

## Deploying a tool to AIDP

1. After running `python setup.py build`, upload `CUSTOM_CODE_TOOLS/<pkg>/<pkg>.zip`
   via **AIDP Console → Tools → Upload custom tool**.
2. Add a Custom Tool node to an agent flow and bind it to the `toolClassName`
   from the package's `tool_config.json`.
3. The conf you filled with `setup.py configure` shows as the defaults; the
   secret fields (PEM key, passphrase, webhook URLs, SMTP/IMAP passwords) must
   be set per-deployment via your secret store.

## Build your own

Two paths:

- **Interactive (recommended):** `python setup.py new-tool` prompts for the
  package name, class name, conf starter shape (minimal / aidp_catalog /
  oci_genai / http / empty), and writes the whole scaffold including a
  working `_execute_tool` stub.

- **Manual:** copy `CUSTOM_CODE_TEMPLATE/`, edit `tool_config.json` and
  `tool_implementation.py`, run `python setup.py build`.

## Notes

- Live-service tools (genai, web, email, object storage, AIDP catalog) need
  their backend reachable; their testing sheets list the prerequisites.
- Test pods have no network. Tools with pip dependencies install at deploy;
  bundle wheels under `wheels/` for offline test pods.
- Secrets are NEVER committed. The wizard always skips fields marked
  `kind: secret` in `_uiHints` (private keys, passphrases, webhook URLs,
  SMTP/IMAP passwords). Provide them per-deployment via your secret store.
